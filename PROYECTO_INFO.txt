EL CENTINELA - SISTEMA DE GESTIÓN DE INVENTARIO PARA RESTAURANTES
================================================================================

DESCRIPCIÓN DEL PROYECTO
========================
El Centinela es una aplicación web desarrollada en Django que permite a los 
restaurantes gestionar eficientemente su inventario de insumos, reduciendo el 
desperdicio y optimizando las compras.

El sistema permite:
- Registrar insumos con identificadores únicos (códigos, códigos QR)
- Rastrear fechas de vencimiento y recibir alertas
- Registrar el consumo diario de insumos
- Calcular promedios de uso y proyecciones de compra
- Visualizar un historial completo de movimientos

CARACTERÍSTICAS PRINCIPALES
================================================================================

1. DASHBOARD (Página Principal)
   - Sección de alertas persistentes con botón X para descartar
   - Alertas se regeneran automáticamente al recargar

2. GESTIÓN DE INSUMOS
   - Crear nuevos insumos

   - Marcar insumos como eliminados (soft delete)
   - Listar todos los insumos activos con paginación

3. REGISTRO DE USO
   - Buscador inteligente con filtro en tiempo real
   - Insumos ordenados automáticamente por fecha de vencimiento

5. PROYECCIÓN DE COMPRA

6. ALERTAS (NUEVO en v2.0)
   - Insumos que vencen en 3 días o menos

MODELOS DE BASE DE DATOS
================================================================================

1. INSUMO
   - id (PK): ID automático
   - nombre: Nombre del insumo (Ej: Pollo, Tomate)
   - id_codigo: Código único o QR del producto
   - cantidad: Cantidad ACTUAL en el inventario
   - cantidad_original: Cantidad ORIGINAL cuando se ingresó (NUEVO en v2.0)
   - unidad: Unidad de medida (kg, g, ml, l, unidad, docena)
   - fecha_entrada: Fecha/hora cuando se agregó al inventario
   - fecha_expiracion: Fecha de vencimiento
   - activo: Boolean para marcar como eliminado sin borrar la BD

   MÉTODOS ÚTILES:
   - dias_para_expirar: Devuelve días faltantes para expiración
   - esta_proxima_a_expirar: True si expira en ≤ 3 días
   - esta_expirado: True si ya pasó la fecha de expiración

2. USOHISTORICO
   - id (PK): ID automático
   - insumo (FK): Referencia al insumo utilizado
   - cantidad_usada: Cantidad consumida
   - fecha: Fecha del uso (automática)
   - notas: Notas adicionales (opcional)

   NOTA: Cuando se crea un UsoHistorico:
         • insumo.cantidad se resta automáticamente
         • Se regeneran las AlertaMensaje si es necesario

3. ESTADISTICASUSO
   - id (PK): ID automático
   - insumo (OneToOne): Referencia única al insumo
   - uso_promedio_diario: Promedio calculado
   - ultima_actualizacion: Última vez que se calculó

   MÉTODO:
   - calcular_promedio(insumo): Calcula promedio de últimos 7 días

4. ALERTAMENSAJE (NUEVO en v2.0)
   - id (PK): ID automático
   - titulo: Título de la alerta (Ej: "Aceite vence pronto")
   - descripcion: Descripción completa
   - tipo: Choices ['vencimiento', 'expirado', 'bajo_stock', 'info']
   - insumo (FK nullable): Referencia al insumo (puede ser NULL)
   - fecha_creacion: Timestamp automático
   - activa: Boolean para marcar como leída/descartada

   FLUJO:
   • generar_alertas_insumo(insumo) crea/recrea alertas
   • generar_todas_alertas() se ejecuta al cargar dashboard
   • Usuario puede descartar con botón X
   • descartar_alerta(request, alerta_id) las elimina de BD

RUTAS Y VISTAS
================================================================================

RUTAS PRINCIPALES:
  
  /                                  → Dashboard (página principal + alertas)
  /insumos/                          → Lista de insumos
  /insumos/crear/                    → Crear nuevo insumo

VISTAS (En views.py):

  dashboard()                        → Genera alertas, renderiza con stats
  InsumoListView                     → List view de insumos (paginada)

  InsumoDeleteView                   → Marcar como inactivo (soft delete)
  UsoHistoricoListView               → Historial agrupado por fecha
  UsoHistoricoCreateView             → Registrar uso (resta cantidad + genera alertas)
  proyeccion_compra()                → Proyección con cálculo de porcentaje
  descartar_alerta()                 → Elimina AlertaMensaje (NUEVO v2.0)
  generar_alertas_insumo()           → Crea alertas para un insumo (NUEVO v2.0)
  generar_todas_alertas()            → Itera todos insumos y genera alertas (NUEVO v2.0)

INSTALACIÓN Y EJECUCIÓN
================================================================================

REQUISITOS:
  - Python 3.8+
  - Django 6.0.6
  - Navegador web moderno

PASOS:

1. Crear y activar entorno virtual:
   $ python -m venv venv
   $ venv\Scripts\activate  # En Windows

2. Instalar dependencias:
   $ pip install django==6.0.6

3. Aplicar migraciones (crear base de datos):
   $ python manage.py migrate

4. Crear superusuario (admin):
   $ python manage.py createsuperuser
   # Ingresar username, email, password

5. Ejecutar servidor de desarrollo:
   $ python manage.py runserver
   
   El sitio estará disponible en: http://127.0.0.1:8000/

6. Panel de administración (Django Admin):
   http://127.0.0.1:8000/admin/
   (Usar las credenciales del superusuario creado)

================================================================================
FLUJO TÍPICO DE USO
================================================================================

ESCENARIO: Restaurant que abre su negocio

1. El gerente accede al dashboard (página principal)

2. Agrega insumos iniciales:
   - Click en "Agregar Nuevo Insumo"
   - Completa: nombre, código QR, cantidad, unidad, fecha_expiracion
   - Repite para todos los insumos iniciales

3. Cada día, registra el consumo:
   - Click en "Registrar Uso"
   - Selecciona el insumo utilizado
   - Ingresa cantidad consumida
   - Agrega notas si es necesario (Ej: "Comida cancelada")

4. Revisa alertas en el dashboard:
   - Ve automáticamente insumos a vencer (< 3 días)
   - Accede a detalles para ver más información

5. Después de 7 días, usa la proyección:
   - Accede a "Proyección de Compra"
   - Ve recomendaciones automáticas
   - Compra exactamente lo que necesita

================================================================================
FUNCIONES ESPECIALES EXPLICADAS
================================================================================

SOFT DELETE (Eliminación Lógica):
  - Al eliminar un insumo, se marca como inactivo (activo=False)
  - El insumo NO se borra de la base de datos
  - El historial se mantiene para análisis
  - No aparece en listas regulares

CÁLCULO DE PROMEDIOS:
  - EstadisticasUso.calcular_promedio(insumo)
  - Toma los últimos 7 días de uso
  - Suma las cantidades y divide entre 7
  - Se usa para proyecciones de compra

PROYECCIÓN DE COMPRA:
  - Usa el uso_promedio_diario
  - Multiplica por 7 para recomendar compra semanal
  - Compara con stock actual
  - Indica si: Comprar urgente / Considerar / Stock OK

================================================================================
TROUBLESHOOTING
================================================================================

ERROR: "No such table: inventario_insumo"
SOLUCIÓN: $ python manage.py migrate

ERROR: "TemplateDoesNotExist"
SOLUCIÓN: Verifica TEMPLATES['DIRS'] en settings.py

ERROR: "ImportError: cannot import name 'inventario'"
SOLUCIÓN: Asegúrate que 'inventario' esté en INSTALLED_APPS

ERROR: Formulario sin estilos
SOLUCIÓN: Verifica que Bootstrap se cargue desde CDN en base.html

ERROR: Timezone issues
SOLUCIÓN: TIME_ZONE debe ser correcto en settings.py
